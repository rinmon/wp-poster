# WordPress 記事予約投稿ワークフロー

## 概要

ユーザーから記事テキストを受け取り、一次情報（実写・公式画像など）を挿入してWordPressに予約投稿する。

## 【厳格ルール】Cursor Composer スタイル

1. **記事のボリューム**：Cursorで書いていた時と同じかそれ以上の長さ（最低1500〜2500文字以上）を確保すること。
2. **写真ポリシー（絶対遵守）**：
   - **一次情報（公式サイト・信頼できる一次ソース）から実写真のみ**使用すること。
   - **AI生成画像の使用は【絶対禁止】**（著作権クリアのためという理由であってもNG）。
3. **写真挿入ルール**：
   - 各セクションに**1〜2枚**の一次情報写真を入れ、適切な位置に配置すること。
   - アイキャッチ（1枚目）は最も象徴的な一次情報写真を指定すること。
   - 画像URLは必ず一次ソースから直接アクセス可能なものを指定すること（Xの場合は必ず非公式API `api.vxtwitter.com` を経由して実在の画像URLを取得する。ダミーのdefaultアイコンは使用しない）。
4. **トーン＆マナー**：
   - 落ち着いたビジネスブログ調（自然で読みやすく、詳細な解説を入れること）。
   - 過度な感情表現や煽るような言葉は使用しない。
5. **記事の構造**：
   - 導入 → 本論（詳細データ・一次情報引用） → 結論 → 関連リンク（最後に記述）
6. **最終出力のマークダウン形式**：
   - 出力はMarkdown形式とし、写真は `![説明](一次ソースのURL)` で挿入すること（`IMAGE_BLOCK`の使用は中止）。

## 手順

### 1. 記事テキストの受け取りとテーマ把握
- テーマに沿ったキーワードと構造を計画。
- 1500〜2500文字以上の深掘り解説を想定。

### 2. 記事ファイルの作成
- `drafts/` フォルダに `（テーマ）.md` ファイルとして記事を保存
  - 1行目: タイトル
  - 2行目以降に本文構成。各見出しの下に `![説明](URL)` を用いて一時情報画像を挿入。
  - 末尾に `**タグ**`, `**カテゴリ**`, `**メタディスクリプション**` を独立して記述。

### 3. 重複チェック（必須）
- 投稿する前に重複がないか確認。
- 実行コマンド: `python3 check_article_duplicates.py --file <作成したドラフト.md>`

### 4. 投稿スクリプト実行
- `drafts/` フォルダにあるファイルを指定して予約投稿。
- 実行コマンド: `python3 api_poster.py --file <作成したドラフトファイル名.md>`

## ファイル形式（drafts/xxx.md）

```markdown
タイトル（1行目）

![アイキャッチ画像の説明](一次ソースのURL)

導入テキスト（背景やサマリー）

## 見出し1（本論）

詳細データや背景情報の解説。少なくとも1500〜2500字以上のボリュームの1部として深い情報提供を行うこと。

![セクション1を象徴する一次ソース画像](一次ソースのURL)

## 見出し2

...テキスト...

## 結論

...テキスト...

## 関連リンク

- [リンクタイトル](URL)

**タグ**
タグ1, タグ2, タグ3, タグ4, タグ5, タグ6, タグ7, タグ8, タグ9, タグ10, タグ11, タグ12

**カテゴリ**
04.【経済・ビジネス】

**メタディスクリプション**
抜粋テキスト（150〜200文字）
```
