# WordPress 記事予約投稿ワークフロー

## 概要

ユーザーから記事テキストを受け取り、AI画像を生成してWordPressに予約投稿する。

## 画像方針（重要）

**記事に画像URLや実際の画像ファイルがない場合、AIで記事内容に合った画像を自動生成する。**
ユーザーへの確認は不要。以下の基準で生成：
- アイキャッチ画像（img_001）：記事テーマを表す建物・施設・企業イメージ
- 本文画像2枚目以降：記事の各セクションに合ったビジュアル（グラフ、業界イメージ等）
- 画像数の目安：見出しセクション数に応じて2〜4枚

## 手順

### 1. 記事テキストの受け取り
- ユーザーから記事テキスト（タイトル・本文・タグ・カテゴリ・メタディスクリプション）を受け取る
- 記事のテーマ・キーワードを把握する

### 2. AI画像生成
- `generate_image` ツールで記事内容に合った画像を2〜4枚生成する（並行実行）
- 生成した画像を `/Users/rinmon/Library/CloudStorage/SynologyDrive-work/0000/BLOG/drafts/` にコピー
  - ファイル名形式: `img_001_xxxx.png`, `img_002_xxxx.png`, ...
  - ソートされた順で最初のファイルがアイキャッチになるため、`img_001` を最も重要な画像にする

### 3. 記事ファイルの作成
- `drafts/` フォルダに `.md` ファイルとして記事を保存
  - 1行目: タイトル
  - 本文中に画像プレースホルダー `[IMG_2]`, `[IMG_3]` を適切な位置に挿入（IMG_1はアイキャッチ）
  - 末尾に `**タグ**`, `**カテゴリ**`, `**メタディスクリプション**` セクション

### 4. 投稿スクリプト実行
// turbo
```bash
cd /Users/rinmon/Library/CloudStorage/SynologyDrive-work/0000/BLOG && python3 api_poster.py
```

### 5. 結果確認
- 投稿成功時: Post ID・スケジュール日時を確認しユーザーに報告
- 投稿失敗時: エラーメッセージを確認して対処

## ファイル形式（drafts/xxx.md）

```
タイトル（1行目）

本文（マークダウン形式）

[IMG_2]

## セクション2

本文続き

[IMG_3]

**タグ**
タグ1, タグ2, タグ3

**カテゴリ**
02.【経済・ビジネス】

**メタディスクリプション**
抜粋テキスト（150〜200文字）
```
